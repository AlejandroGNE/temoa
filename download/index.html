Download and Installation

The Temoa energy system model is implemented with Pyomo, which is in turn written in Python. Consequently, Temoa will run on Linux, Mac, Windows, or any operating system that Pyomo supports.

There are several open source software elements required to run Temoa. Because the licenses vary between them, we cannot create a single, customized installation package. However, the installation process does not take very long. Here are the pieces you will need to install, and it is best to install them in the order given below:

  1. Python. Temoa requires at least Python v2.7. For our own development, we currently use a Python distribution called Anaconda, which is available for free through Continuum Analytics. Another popular distribution is Enthought Canopy (the 'Express version is freely available). The main advantage of using a Python distribution is that it automatically curates a large number of Python packages (e.g., numpy, scipy, matplotlib), thus eliminating the need for ad hoc Python package management. If you install Anaconda for use with Temoa, make sure that during installation it gets added to your PATH.

  2. Pyomo. Temoa employs APIs from PYthon Optimization Modeling Objects (Pyomo), which was developed at Sandia National Laboratories. This is the software that allows us to implement the algebraic formulation of Temoa in Python without using commercial algebraic modeling languages, such as GAMS or AMPL. Pyomo is installed with two simple commands that utilize the Python 'pip' utility: 
    $ pip install pyomo 
    $ pip install pyomo-extras
    For more information on Pyomo installation, see these instructions.

  3. A linear optimization solver. We recommend GLPK as it is GPL software and thus free. A couple notes on GLPK installation. If you follow the link above, it points you to the download of a tarball that must be extracted and compiled in C. The procedure is relatively straightforward if you are running linux, as a c compiler is already installed. If using Windows, you can download a pre-compiled version here. If you plan to run Temoa with large datasets and/or conduct uncertainty analysis, you may want to consider commercial linear solvers such as CPLEX or Gurobi. Both offer free academic licenses. Note that whatever solver you choose, it must be added to your PATH or Pyomo won't be able to find it.

  4. The Temoa model. There are a couple of options for obtaining and running Temoa. The first option is to download temoa.py and example_data_sets.zip. The 'temoa.py' file is actually a zip archive, but you can run it from the command line as an executable. (To see the actual model files, change the extension from '.py' to '.zip' and then simply unzip the archive.) The second option creates a local copy of the source code and input data archived in our GitHub repository. This is a two step process: first install git and then 'clone' the repository. Under Linux, git can be installed through the default package manager. Git for Windows and Mac can be downloaded here. To clone the Temoa repository, navigate to the directory where you want the model to reside and type the following from the prompt: 
    $ git clone https://github.com/TemoaProject/temoa/
    Note that cloning the repository will supply the latest version of the code, while the downloads described above may reference older versions of the code with potentially less functionality.

The installation procedures above are meant to be generic and should work across different platforms. Nonetheless, system-specific ambiguities and unexpected conditions inevitably arise. Please use the Temoa forum to ask for help.

Running Temoa

If you chose to download the zip archive and sample datasets, Here's how to run the zip file as an executable: 

$ pyomo temoa.py /path/to/dat/file
The command above invokes Pyomo directly to instantiate the model and produces basic output. You can instead instantiate the model with Python: 
$ python temoa.py /path/to/dat/file
This version enables enhanced functionality, including formatted results as well as the capability to make graphviz plots and to run stochastic optimization with pysp. To see the options available for running Temoa, type the following at the command prompt: 
$ python temoa.py --help
You will be presented with a set of flags that can be used to enable different pieces of model functionality at runtime.

If you instead clone the Temoa GitHub repository, the commands to run the model are analogous to the ones above, except the model folder is referenced rather than the zipped executable: 

$ pyomo temoa_model/ /path/to/dat/file
or 
$ python temoa_model/ /path/to/dat/file

For more information, see the Temoa user manual.
